# Daniel Arella — User Journey

Define cómo vive un lector dentro del sitio.

No es un diagrama. Es una lista de recorridos reales. Sirve para validar navegación y copy y evita pantallas inútiles y pasos innecesarios.

**Referencia:** `arquitectura-editorial.md`, `ui-copy-sheet.md`

---

## 1. Lector llega y lee

Inicio → Poema destacado o Libro activo → Leer → Volver al archivo

---

## 2. Lector explora la obra

Archivo → Filtrar por tema o periodo → Abrir poema o ensayo → Ver el libro al que pertenece

---

## 3. Lector se acerca al autor

Sobre el autor → Correspondencia o Contacto

---

## 4. Lector quiere un taller

Talleres → Abrir próximo → Contactar o inscribirse
