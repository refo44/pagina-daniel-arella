# Daniel Arella — Plan Maestro de la Plataforma de Autor

**Versión 1.6**

La identidad corporativa definida en `02-identidad-corporativa` es una capa visual y editorial ya cerrada.

Este documento define algo más profundo: el sistema que permite que la obra de Daniel Arella exista, se lea, se recorra y perdure como un cuerpo vivo.

Este sitio no es una web.  
Es un territorio editorial.

El problema que resuelve este plan no es estético. Es ontológico:

- ¿Qué es un poema aquí?
- ¿Qué es un libro?
- ¿Cómo se atraviesa una obra sin perderla?

---

## Principios operativos

Los grandes sitios de autor no funcionan como portafolios ni como tiendas. Funcionan como espacios vivos de obra.

| Principio | Qué significa |
|-----------|----------------|
| **Espacio vivo** | El sitio cambia, rota y respira. No es un archivo muerto ni una portada fija. |
| **Una sola tesis en el home** | El home responde una sola pregunta: qué obra está viva ahora. |
| **La marca es atmósfera** | Tipografía, ritmo y silencio pesan más que los ornamentos. |
| **Relación directa** | El lector se conecta con el autor, no con plataformas. |
| **Prueba social silenciosa** | Premios y editoriales existen, pero no gritan. |

Este plan no imita esos sitios. Los convierte en arquitectura.

---

## Las ocho capas

La plataforma está diseñada en capas que no se contaminan entre sí. Cada capa cumple una función precisa.

### Capa 1. Identidad

Define cómo se ve y cómo se siente el sistema.

Incluye:

- Tipografía
- Paleta
- Ritmo
- Uso del blanco
- Oscuridad y luz
- Jerarquía editorial
- Voz visual

Esta capa ya está cerrada en `02-identidad-corporativa`.

### Capa 2. Arquitectura de información

Define qué existe.

Un sitio de autor serio necesita:

| Sección | Función |
|---------|---------|
| Inicio | Presente vivo de la obra |
| Poesía | Libros y poemas |
| Ensayo | Pensamiento y crítica |
| Narrativa | Relatos y cuentos |
| Talleres | Actividad pedagógica |
| Archivo | Biblioteca completa |
| Sobre el autor | Biografía intelectual |
| Publicaciones | Libros y PDFs |
| Contacto | Prensa, editores, lectores |

El archivo no es un feed. Es una biblioteca navegable.

### Capa 3. Modelo de contenido

Define qué es cada cosa.

| Tipo | Qué representa |
|------|----------------|
| Poem | Un poema individual |
| Book | Una obra cerrada |
| Essay | Pensamiento |
| Story | Narrativa |
| Workshop | Taller o curso |

Un poema no es un post. Un libro no es una página. Una obra no es un blog.

Los ciclos, series y conjuntos se narran dentro de los libros o mediante enlaces. No existen como tipos técnicos independientes.

Las taxonomías mínimas son: Tema, Periodo, Forma (solo si se usa). El idioma lo gestiona el sistema de traducción.

### Capa 4. Gramática de navegación

Define cómo se recorre la obra.

Incluye:

- Breadcrumbs por libro
- “Del mismo libro”
- “Textos relacionados”
- Archivo navegable por: Tipo, Tema, Periodo, Libro

El lector siempre sabe dónde está dentro del cuerpo de la obra.

### Capa 5. Estrategia de publicación

Aquí decide Daniel. El sistema solo lo permite.

Incluye:

- Publicar por libros
- Abrir o cerrar etapas
- Publicar PDFs
- Mostrar obra inédita
- Activar o desactivar secciones
- Definir qué está vivo ahora en el home
- Relación por email bajo tono autoral

El sistema soporta cualquier ritmo sin romperse.

### Capa 6. Sistema editorial

Define cómo se escribe para este territorio.

Incluye:

- Longitud
- Notas
- Citación
- Fragmentos
- Voz

La plataforma no dicta estilo. Garantiza coherencia.

### Capa 7. Infraestructura

La base invisible que protege la obra:

- Hosting estable
- Backups
- SSL
- Cache
- Control de versiones
- Staging

Una obra sin backups no es una obra. Es una pérdida en potencia.

### Capa 8. Legal y derechos

Protege la obra:

- Copyright
- Licencias
- Permisos de descarga
- Contacto editorial

## Lo que este sistema permite

Sin tocar código, Daniel puede:

- Publicar un poema o un libro
- Traducir textos
- Cambiar de etapa
- Activar talleres
- Reordenar su archivo
- Cambiar el foco del home

El sistema no se rompe.

---

## Venta y suscripción sin romper la obra

Este sistema sí permite:

- Vender libros
- Vender talleres
- Recibir suscriptores

Pero nunca como marketing. Se expresa como:

- “Adquirir el libro”
- “Inscribirse al taller”
- “Recibir textos”

La obra es el centro. La transacción es un gesto editorial.

---

## Lo que este sistema no es

No es:

- Un funnel
- Una landing
- Una tienda
- Un CMS inflado

Es un territorio de obra que puede sostener dinero sin perder dignidad.
